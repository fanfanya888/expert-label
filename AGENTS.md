## 你的角色
你正在帮助构建一个早期阶段的专家标注平台。
你的任务是实现一个扎实、最小化、可运行的工程基础骨架。

## 当前阶段
当前处于 Phase 1B（加强版第一阶段）。

目标不是构建完整产品，
而是构建一个结构清晰、可运行、可扩展的项目底座。

## 当前阶段的核心目标
1. 创建清晰的项目目录结构
2. 使用 Docker Compose 搭建 PostgreSQL
3. 搭建 FastAPI 后端
4. 通过环境变量实现配置管理
5. 添加 SQLAlchemy Base 和数据库会话管理
6. 初始化 Alembic 并支持迁移
7. 添加两个基础模型：User 和 Project
8. 添加统一响应结构
9. 添加最小插件架构骨架
10. 保持整体实现简单、清晰、可运行

## 当前阶段不要做的事情
- 不要拆成微服务
- 不要添加 Redis、Celery、MinIO、Nginx、消息队列
- 不要实现完整认证或 RBAC
- 不要接入真实 LLM
- 不要构建复杂前端页面
- 不要过度设计抽象
- 不要生成大量推测性的业务代码

## 工程原则
- 优先实现最小可运行版本
- 优先保证清晰而不是炫技
- 优先保证结构稳定而不是过早复杂化
- 保持文件职责单一
- 避免不必要的依赖
- 使用常规、易懂的命名
- 细节不明确时，做最简单且合理的实现，但不要超出当前阶段范围
